def GameTools.startClass (   module = "myModule",
  name = "myClass",
  parent = "",
  attributes = [],
  directory = "" 
)

A method to create the start for a python class structure with getters and setters for all the attributes.


Parameters
moduleThe python module (a.k.a. file) to write the object to, it writes in append mode so it will not delete objects and methods already in the module.
nameThe name for this class
parentThe class the new class will inherit from if applicable
attributes[]A list of strings representing all the desired attributes for the class. Each element in this list will be added to the init constructor and have a getter and setter method generated.
directoryThe directory to write the objects to, will be created if it does not already exist

A method to generate the method header and attach it to an object or module.


Currently this is only set up to work if there is only one single class per file or a module of non-class methods to import (the new function is added at the bottom of the file, and if there are any classes in the module it will be indented)# Currently this is only set up to work if there is only one single class per file or a module of non-class methods to import

Parameters
nameA name for the function to be added to an object for either a 2d or 3d game
moduleThe module (a.k.a. python file) to add the function into, if omitted it will be added directly to the module containing the main game object
parametersA list of strings representing the parameters that will be needed for the method

A method to generate a set up for a new task to be added to the task manager in the Panda3D engine.


Currently this is only set up to work if there is only one single class per file or a module of non-class methods to import (the new function is added at the bottom of the file, and if there are any classes in the module it will be indented)

Parameters
nameA name for the task to be added to the Panda3D task manager
